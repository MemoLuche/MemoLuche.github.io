Project Overview

Challenge

Design and develop a complete event management system for wedding planners in just 2 weeks, handling complex inventory tracking, service management, and multi-user coordination.

Solution

Built a full-featured web application with role-based access, real-time inventory management, automated notifications, and comprehensive event tracking using modern web technologies.

Impact

Delivered a production-ready system managing 40+ events, tracking 100+ inventory items, and handling multiple concurrent users with a clean, intuitive interface.

Key Architectural Decisions

  • MVC Pattern: Separation of concerns with modular file structure
  • PDO with Prepared Statements: Protection against SQL injection
  • Session-Based Auth: Secure user authentication and role management
  • Responsive Design: Mobile-first approach with Bootstrap
  • Environment Detection: Automatic switching between local and production configs

Database Design

Designed a robust relational database with 13 interconnected tables supporting complex relationships and data integrity.

Core Tables

  • usuario - User accounts and roles
  • eventos - Event details and scheduling
  • inventario - Inventory items catalog
  • servicios - Available services

Junction Tables

  • evento_inventario - Event-inventory relationships
  • evento_servicio - Event-service assignments
  • evento_cronograma - Event timelines

Tracking Tables

  • operaciones_inventario - Inventory history
  • operaciones_servicios - Service operations
  • notificaciones - User notifications
  • historial_notificaciones - Notification logs

Challenges & Solutions

Complex Data Relationships

Challenge: Managing multiple interconnected entities (events, inventory, services, users) with complex relationships.

Solution: Designed normalized database schema with proper foreign keys and junction tables. Implemented PDO with prepared statements for data integrity.

Tight Timeline

Challenge: Delivering a full-featured system in just 2 weeks with a team of 5 students.

Solution: Implemented agile methodology with daily standups, clear task division, and parallel development. Used Bootstrap to accelerate UI development.

Real-Time Inventory Tracking

Challenge: Ensuring inventory availability across multiple concurrent events without conflicts.

Solution: Implemented transaction-based inventory operations with status tracking and automatic availability calculations.

Multi-User Coordination

Challenge: Team members working on different features simultaneously without code conflicts.

Solution: Used Git branching strategy with feature branches, regular merges, and code reviews. Established clear module boundaries.
