MAKA OWS 2.0

Next-Generation Event Management Platform with Microservices Architecture

January 2026 - Present 7 Microservices Enterprise-Grade

About This Project

A complete rebuild of the original MAKA OWS event management system, transitioning from a monolithic PHP architecture to a modern microservices-based platform using enterprise-grade technologies.

This project is being developed for Proyectate 2026 (UAQ's academic exhibition) while building a commercial-ready product for the event management market.

Project Overview

Vision

Transform event management with a scalable, modern platform that handles everything from inventory tracking to financial management, built with enterprise-grade architecture.

Goals

Create a commercial-ready product for Proyectate 2026 exhibition while building a real-world portfolio piece that demonstrates mastery of modern software architecture.

Impact

Replacing monolithic PHP architecture with scalable microservices, enabling horizontal scaling, independent deployments, and enterprise-level reliability.

Backend: Microservices Architecture

The system is designed as a collection of independent, loosely-coupled services that communicate through well-defined APIs. Each service handles a specific business domain and can be developed, deployed, and scaled independently.

Auth Service

Authentication, authorization, user management, JWT token generation, role-based access control (RBAC)

Event Service

Event CRUD operations, status management, assignments, scheduling, budget tracking, client relations

Inventary Service

Inventory management, stock tracking, item assignments to events, availability checking, categories

Services Service

Service catalog, provider management, service assignments, pricing, availability scheduling

Notification Service

Email notifications, alerts, reminders, notification templates, delivery tracking

Email Service

Transactional emails, email templates, delivery management, bounce handling

API Gateway

Request routing, authentication middleware, rate limiting, CORS handling, response transformation
